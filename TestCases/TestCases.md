# Тест-кейсы для TodoList

## TC-01: Создание нового пользователя

**Описание:** Проверка корректного создания нового пользователя.

**Предусловия:** Приложение запущено. Пользователь не авторизован.

**Действия:**
1. Запустить приложение.
2. На запрос "Войти в существующий профиль? [y/n]:" ввести `n`.
3. Ввести логин: `Kirill`.
4. Ввести пароль: `12345`.
5. Ввести имя: `Кирилл`.
6. Ввести фамилию: `Десятун`.
7. Ввести год рождения: `2007`.

**Ожидаемый результат:**
- Выводится сообщение "Профиль создан. Добро пожаловать, Кирилл!".
- Пользователь автоматически авторизован.
- Данные сохранены в файл `data/profiles.csv` (зашифрованы).

**Скриншоты:**
- `Screenshots/Скрин1` – после завершения регистрации

## TC-07: Добавление многострочной задачи (--multiline)

**Описание:** Проверка ввода задачи с переносами строк.

**Действия:**
1. Ввести `add --multiline`.
2. Ввести первую строку: `Строка 1`.
3. Ввести вторую строку: `Строка 2`.
4. Ввести `!end`.

**Ожидаемый результат:**
- Задача добавлена с текстом "Строка 1\nСтрока 2".
- При `read <индекс>` отображается полный текст с переносами.

## TC-16: Сохранение и загрузка данных

**Описание:** Проверка, что после перезапуска приложения задачи сохраняются.

**Предусловия:** Добавлены задачи, изменены статусы.

**Действия:**
1. Закрыть приложение (команда `exit`).
2. Запустить приложение снова.
3. Авторизоваться под тем же пользователем.
4. Ввести `view`.

**Ожидаемый результат:**
- Все задачи и их статусы совпадают с состоянием до закрытия.
